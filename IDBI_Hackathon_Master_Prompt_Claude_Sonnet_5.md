# Hackathon Master Prompt (C.R.A.F.T.) for Claude Sonnet 5

## Goal

Win the IDBI Innovate 2026 Hackathon for **Problem Statement 2 --
Prospect Assist AI**.

# C --- Context

You are assisting a team of experienced enterprise AI engineers
participating in the IDBI Innovate 2026 Hackathon.

Before producing any output, ingest every uploaded document in full.

Priority order: 1. Prototype Submission Deck (PPT template) 2.
Orientation Session transcript 3. Problem Statement Explainer & AMA
transcript 4. Any additional documents uploaded later

Treat these as the single source of truth. Never contradict them.

Primary objective:

Design a startup-worthy, production-ready, AWS-first Agentic AI platform
that enables IDBI Bank to identify high-quality retail lending prospects
by combining:

-   Transaction intelligence
-   Behavioural analytics
-   Repayment capacity estimation
-   Intent prediction
-   Explainable AI
-   Human-in-the-loop underwriting

The solution must target: - \>30% conversion - Better underwriting -
Higher operational efficiency - Practical deployment inside IDBI

Assume: - 24-hour hackathon - Working prototype - Synthetic data
initially - AWS preferred - APIs available later - Banking regulations
respected - Judges value innovation, scalability, adaptability,
production readiness and demo quality.

# A --- Action

## Phase 0

Read all uploaded files.

Extract: - hidden judging hints - terminology - business expectations -
mandatory constraints - APIs - AWS preferences - demo expectations -
production readiness signals

Create a concise knowledge base.

## Phase 1

Deeply analyse the lending problem.

Identify: - root causes - customer pain points - banker pain points -
operational inefficiencies - opportunities competitors miss

## Phase 2

Generate 20 radically different ideas.

Avoid generic ML lead scoring.

## Phase 3

Score every idea on: - Innovation - Business Value - Scalability -
Practicality - Demo impact - Patent potential - Startup potential - AI
sophistication - Ease within 24 hours

Discard weak ideas.

## Phase 4

Merge the strongest concepts into one differentiated product.

Explicitly explain why judges would remember it.

## Phase 5

Design the complete product.

Include: - Personas - User journeys - KPIs - Business workflow -
Customer lifecycle - Success metrics

## Phase 6

Design the Agentic AI system.

Define every agent.

Suggested agents: - Orchestrator - Prospect Intelligence - Behaviour
Agent - Repayment Capacity Agent - Intent Detection Agent - Income
Estimation Agent - Fraud Agent - Explainability Agent - Compliance
Agent - Recommendation Agent - Conversation Agent

For every agent provide: - Inputs - Outputs - Tools - Memory - Prompts -
Failure handling

## Phase 7

Design AWS architecture.

Prefer: - Bedrock - Lambda - API Gateway - ECS/EKS - DynamoDB - Aurora -
S3 - EventBridge - Step Functions - OpenSearch - Kinesis - CloudWatch -
Cognito - Secrets Manager

Explain why each service is chosen.

## Phase 8

Design: - APIs - Database schema - Event flow - Sequence diagrams -
HLD - LLD

## Phase 9

Generate realistic synthetic banking datasets.

Include: - Salaried - Self-employed - Gig workers - Existing borrowers -
High intent - Window shoppers - Good and poor repayment behaviour

## Phase 10

Create explainable AI outputs.

Every recommendation must include: - Confidence - Features influencing
score - Risks - Human-readable reasoning

## Phase 11

Design prototype.

Include: - Screens - Wireframes - Demo flow - User interactions -
Dashboard - Loan officer view - Customer journey

## Phase 12

Complete every slide of the uploaded PPT template.

Maintain the exact order and tailor the content to the template.

## Phase 13

Generate: - 3-minute demo script - 5-minute presentation - Elevator
pitch - Judge Q&A - Technical Q&A - Business Q&A

## Phase 14

Critique everything.

Challenge assumptions.

Identify: - weaknesses - regulatory risks - scalability bottlenecks - AI
hallucination risks - privacy issues - adoption risks

Improve accordingly.

## Phase 15

Deliver the final polished submission.

# Non-negotiable Rules

-   Do NOT produce ordinary hackathon ideas.
-   Continuously compare against what average teams would build.
-   If an idea feels obvious, improve it.
-   Optimize for startup potential.
-   Optimize for production deployment.
-   Every AI decision must be explainable.
-   Respect banking privacy and regulatory principles.
-   Human approval remains mandatory for lending decisions.
-   Explicitly identify assumptions.
-   Ask clarifying questions only if absolutely necessary; otherwise
    proceed with justified assumptions.

# Success Criteria

> "This is not just a hackathon project. This could become an actual
> IDBI product."
